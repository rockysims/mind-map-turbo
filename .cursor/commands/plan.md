# Write A Milestone Plan

Use this command when the user asks to write, revise, or start executing a
roadmap milestone plan.

First read and follow `@.cursor/rules/plans.mdc`. Treat it as the source of
truth for plan structure, task fields, wave planning, and execution workflow.

Then:

1. Read the relevant milestone doc end-to-end.
2. Read `@docs/roadmaps/PLAN_TEMPLATE.md`.
3. If creating a new plan, copy the template into the active roadmap's
   `plans/` directory using today's date and a short slug.
4. Resolve every milestone open question. If a question genuinely needs user
   input, leave an `ASK USER:` marker and surface it in chat.
5. Keep implementation work out of scope until the plan is reviewed, unless
   the user explicitly asks to execute the plan.

If the user did not provide a milestone, ask which milestone to plan before
editing files.
